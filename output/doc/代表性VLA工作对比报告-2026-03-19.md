# 代表性 Vision-Language-Action 工作对比报告

## 从通用操作 VLA 到类人全身 Loco-Manipulation 的阶段性演化

2026年3月19日

## 摘要

本文基于本地文献库中 11 篇代表性论文，对当前 Vision-Language-Action (VLA) 研究的发展路径进行系统比较，并辅以 4 篇补充参照工作作为背景支撑。本文的基本判断是：VLA 的演进已不再是单纯依靠更大模型或更多数据的线性扩展，而是围绕四个核心矛盾逐步展开，即动作表示是否适合连续控制、空间表征是否足以支撑真实操作、长时任务是否需要显式记忆，以及具身执行约束是否必须通过分层控制与系统协同来处理。以 OpenVLA 为代表的早期工作确立了开放式通用 VLA 的基本范式；以 pi0、OpenVLA-OFT、HybridVLA 为代表的工作则不断重塑动作生成与后训练配方；以 SpatialVLA、BridgeVLA、InternVLA-M1 为代表的工作把空间建模从隐式能力提升为显式训练目标；以 MEM 为代表的工作进一步将长时记忆模块化；而 LeVERB、WholeBodyVLA、Psi0 则表明，当研究目标转向 humanoid whole-body loco-manipulation 时，仅靠端到端 VLA 主干已经不足，必须同时引入 latent interface、专用低层控制器、分阶段数据配方与部署时稳定化机制。因此，相比旧工作，新工作的实质性差异主要体现为：空间、动作、记忆与执行约束被逐步显式化、结构化与系统化。

## 一、引言与选题范围

本文的目标不是简单罗列近期 VLA 论文，而是从研究演化的角度回答三个问题：第一，VLA 这一路线到底是如何从通用桌面操作逐步走向复杂具身控制的；第二，新工作相对旧工作究竟新增了什么，而不是仅仅“性能更高”；第三，哪些差异具有结构性意义，值得在后续研究中优先关注。

为此，本文选取 11 篇主比较论文作为主体样本，覆盖从 OpenVLA 到 Psi0 的关键节点，并以 RDT-1B、VLA-R1、LingBot-VLA、ABot-M0 作为补充参照。选择标准主要有四点：其一，论文必须构成一条可追踪的技术主线；其二，论文在方法层面提出了相对清晰的新变量，而非单纯扩大训练规模；其三，论文在公开讨论中具有较高代表性；其四，论文本身在真实机器人、仿真基准或系统落地方面给出了可比较的证据。

从整体上看，这 11 篇论文并不属于同一条窄路线，而是共同刻画了 VLA 研究由浅入深的四个转向：其一，从“把动作离散成 token”走向更适合连续控制的 expert、flow、diffusion 与 latent interface；其二，从隐式空间理解走向显式的 3D / spatial grounding；其三，从短时上下文走向显式的长时记忆；其四，从单臂操作走向 humanoid whole-body loco-manipulation 时，研究对象从单一 policy 逐渐扩展为完整系统。

## 二、总体发展脉络：VLA 的四次重心转移

第一阶段可以视为通用操作 VLA 范式的建立。以 OpenVLA 为代表的工作解决的是“是否能够把开放式视觉-语言模型直接转化为可复用的机器人策略”这一问题。其历史意义主要在于建立开放、可训练、可微调、可部署的基线，从而使 VLA 从封闭展示系统变成社区可复现的研究对象。

第二阶段的核心问题转向动作接口是否合理。pi0、OpenVLA-OFT、HybridVLA 共同说明，早期 token-based VLA 虽然在统一建模上很自然，但并不一定适合高频、连续、双臂或长时控制。由此，研究重心开始从“是否能统一建模”转移到“统一之后如何更好地产生动作”。这一阶段最关键的变化，是动作生成从单一路径演化为连续 expert、chunk 预测、混合 AR-diffusion 乃至 latent interface。

第三阶段的核心问题转向空间表征与推理。SpatialVLA、BridgeVLA、InternVLA-M1 反复指向同一个事实：VLA 的语义理解能力并不自动等价于可靠的空间操作能力。于是，空间信息的角色从以往的隐式能力逐渐升级为显式设计目标。具体表现为：有的工作显式重构 observation/action token 的空间结构，有的工作重新设计 3D 输入输出接口，有的工作则进一步把 spatial grounding 提升为独立预训练任务。

第四阶段则是记忆与类人扩展。MEM 说明，当任务从短时操作走向多阶段、部分可观测、需要中间状态追踪的长时任务时，仅靠增加上下文长度并不足够，必须显式设计多尺度记忆机制。与此同时，LeVERB、WholeBodyVLA、Psi0 进一步表明，一旦研究目标转向 humanoid whole-body loco-manipulation，问题不再只是动作生成，而是高层语义、低层稳定控制、训练数据来源、teleoperation 方式与部署时控制平滑之间的联合优化问题。

## 三、主线一：通用操作 VLA 从开放基线到连续控制

OpenVLA 的代表性并不在于它第一次提出 VLA 概念，而在于它把这一范式真正开放化。它沿用了“图像与语言进入 VLM、动作以 token 形式输出”的基本思路，但用一个社区可复现实验栈将其固定下来。换言之，OpenVLA 的贡献更像是把“VLA 作为概念”转化为“VLA 作为公共基线”。如果没有这一步，后续关于动作表示、空间建模和后训练配方的比较其实难以成立。

相较之下，pi0 的关键推进不在于更大的 backbone，而在于对动作接口的根本性重写。它保留了 VLM 作为高层语义 backbone 的角色，但不再坚持所有动作都必须经由离散 token 产生，而是引入了 flow-based action expert 来处理连续动作 chunk。此处的变化非常重要：VLA 不再被理解为“把动作当作语言预测”，而开始被理解为“以 VLM 提供语义与视觉先验、再由专门的连续控制模块产生动作”的复合系统。对于双臂或复杂精细操作，这一变化几乎是决定性的。

OpenVLA-OFT 则从另一个方向修正了同一问题。它没有更换基础模型，而是系统分析后训练时的动作解码方式、动作表示和目标函数，最终给出 parallel decoding、action chunking、continuous action representation 与 L1 regression 的 OFT 配方。它的重要性在于提醒研究者：很多所谓“模型上限”的问题，其实是适配接口与训练配方的问题，而不一定是 backbone 本身的能力不足。换言之，OFT 证明了 VLA 研究已进入“配方工程学”阶段。

若把这三篇论文放在一条线上看，可以清楚地看到一个趋势：OpenVLA 建立了开放式 VLA 的统一范式，pi0 则把动作生成从语言式 token 预测推进到更适合 dexterous continuous control 的 expert 机制，而 OFT 进一步说明，即便在既有 backbone 下，只要重写 fine-tuning 接口，也可以显著改写性能边界。因此，这一主线的核心演化并非“模型越做越大”，而是“动作接口越来越贴近真实机器人控制的物理需求”。RDT-1B 作为补充参照，也强化了这一判断：在双臂和连续控制场景中，diffusion / continuous action 族方法并不是可选增强，而是在很多任务上更合适的基础设定。

## 四、主线二：空间从隐式能力转向显式训练目标

SpatialVLA、BridgeVLA 与 InternVLA-M1 共同构成了“空间显式化”这一主线，但三者的推进层次并不相同。SpatialVLA 仍保留了典型的 autoregressive VLA 框架，只是在 observation 和 action 两侧显式加入了空间结构。例如，Ego3D position encoding 试图让图像特征带有更明确的三维位置信号，Adaptive Action Grids 则让动作 token 本身具有更强的空间可解释性。它代表的是一种“在既有 VLA 范式内部改良空间表征”的路线。

BridgeVLA 更进一步。它并不满足于在 token 内部注入空间信息，而是重新审视 VLM 原生接口与 3D 操作任务之间的对齐关系。其核心观点是：许多 3D VLA 失败，不是因为没有用到 3D，而是因为 3D 信息的注入方式破坏了 VLM 原有的输入输出结构。因此，BridgeVLA 使用多视角投影和 heatmap 中介，将 3D 感知重新翻译回 VLM 更擅长的 2D 对齐接口。这一点非常关键，因为它表明“空间增强”未必要把模型推向更复杂的 3D token 化，相反，合理的结构对齐可能比更复杂的 3D 表示更重要。

InternVLA-M1 则把空间问题进一步上升为训练目标本身。与 SpatialVLA 和 BridgeVLA 相比，它不再主要关心“如何把空间信息注入现有 VLA”，而是主张在 action post-training 之前先独立学习一个 spatial grounding prior。换言之，空间不再只是 feature engineering 的附属变量，而成为决定 where to act 的核心认知层。由此，空间 grounding 从“增强模块”变成了“前置先验”。这一点解释了为何 InternVLA-M1 在 instruction-following、cluttered pick-and-place 以及长时规划中都体现出更一致的提升。

若进一步结合 VLA-R1 这一补充参照，可以看到空间问题又与 reasoning 问题重新汇合。VLA-R1 通过显式的 chain-of-thought 与 verifiable reward 强化 affordance 与 trajectory reasoning，说明空间能力最终不仅是视觉几何问题，也是中间推理链条能否被优化的问题。因此，从 SpatialVLA 到 BridgeVLA，再到 InternVLA-M1 和 VLA-R1，VLA 社区对空间的理解发生了清晰变化：从“空间是模型可能学会的能力”，变为“空间必须以表示、接口、预训练任务乃至推理监督的方式被显式组织起来”。

## 五、主线三：记忆的显式化与长时任务扩展

MEM 的出现意味着 VLA 研究开始正面处理一个早期工作往往回避的问题：机器人在多阶段任务中需要记住什么，以及这些记忆是否可以由统一上下文窗口自然承载。早期许多 VLA 系统实际上默认任务较短，或者默认当前帧足以代表决定动作所需的大部分信息。然而，一旦任务涉及 recipe tracking、drawer state、object counting、occlusion compensation 或 in-context adaptation，这一假设就会迅速失效。

MEM 的重要贡献在于，它没有把记忆理解为简单的“更多历史帧”，而是明确区分了短时视觉记忆与长时语义记忆。前者以 video encoder 的方式保留对局部运动与遮挡状态的高密度感知，后者则以 language memory 的形式压缩远距离语义事件。也就是说，MEM 认为机器人记忆本身具有多尺度、多模态特征，不能由单一表征统一承担。

从发展路径上看，MEM 相比旧工作的增量不是“在原模型后面再接一个 memory 模块”这么简单，而是把长时任务能力从隐式上下文依赖变成了架构层面的设计问题。这一转变与空间显式化的逻辑非常相似：一项能力一旦被证明对任务成败具有结构性影响，它最终就会从隐式副产物演化为显式模块。对于给导师看的总结而言，这一点值得特别强调，因为它说明当前 VLA 的演化已经进入“能力模块化”阶段，而不再是单一 backbone 规模竞争。

## 六、主线四：从桌面操作走向 humanoid whole-body loco-manipulation

LeVERB、WholeBodyVLA 与 Psi0 共同揭示了一个重要事实：当研究目标从单臂桌面操作转向 humanoid whole-body loco-manipulation 时，传统的 VLA 假设会整体失效。其原因并不难理解。桌面操作更多依赖上肢末端执行器与局部场景互动，而 humanoid loco-manipulation 则同时要求机器人在大空间内完成移动、转身、侧移、下蹲、稳定支撑和上肢操作。此时，高层语义决策与低层身体稳定控制之间的时间尺度与控制要求明显分化，单一端到端策略很难同时兼顾。

LeVERB 的贡献在于较早地把这一分化显式化。它提出 latent verb 接口与双过程层级结构：高层视觉-语言模型输出 latent command，低层 whole-body controller 以更高频率执行。相较于旧工作的显式速度或末端位姿接口，LeVERB 试图保留更丰富的语义表达，同时避免高维 whole-body 动作直接由慢速 VLA 逐步回归。换言之，它提出了一个很重要的判断：在 humanoid 场景下，接口设计本身就是决定成败的核心变量。

WholeBodyVLA 则沿着这一方向进一步推进，但把关注点从 latent interface 扩展到了低层 controller 的任务适配。该工作明确指出，普通 velocity-tracking locomotion policy 并不适合 manipulation-aware locomotion，因为后者需要更强的启停稳定性、更小的姿态误差和更好的对齐能力。因此 WholeBodyVLA 的关键不只是 latent pretraining，而是 latent VLA 与 loco-manipulation-oriented RL controller 的协同设计。与 LeVERB 相比，它更清楚地展示了 whole-body 场景下“高层策略再好，也必须由任务对齐的低层控制器来兑现”的系统事实。

Psi0 又在此基础上进一步强调 staged training 的必要性。其核心观点是，人类 egocentric video 与 humanoid robot data 之间存在显著 embodiment gap，简单混合并不会自然带来更好的类人控制。于是它把学习过程拆分为三个阶段：先用人类视频预训练 VLM backbone 获取一般化视觉-动作语义，再用 humanoid 数据训练 action expert 学习精确关节控制，最后在任务内微调 action expert，并通过 real-time chunking 减少部署时的动作抖动。这说明在 humanoid 全身控制中，训练分解与部署时稳定化机制已经和模型本体同等重要。

因此，如果只用一句话概括这三篇论文相对旧工作的共同增量，那就是：它们都不再把 humanoid whole-body loco-manipulation 视为“更难一点的 manipulation”，而是把它视为一种必须由分层接口、专用低层控制器、特定训练数据来源与系统级部署工程共同支撑的新问题域。这里的差异不是量变，而是问题定义的改变。

## 七、跨论文综合判断：新工作相对于旧工作的主要差异

第一，动作接口正在从统一离散 token 预测走向多样化和任务对齐化。OpenVLA 的动作 token 方式在建立统一范式时非常有效，但当控制频率、动作维度与接触精度要求提高时，pi0、RDT-1B、HybridVLA 以及 humanoid 系统中的 latent interface 都说明，连续 expert、diffusion、flow 乃至 latent command 更适合真实控制问题。

第二，空间能力已经从隐式属性转向显式训练目标。旧工作往往默认 VLM 的多模态语义能力能够自然迁移到空间操作，但 SpatialVLA、BridgeVLA、InternVLA-M1 与 VLA-R1 都表明，空间关系、几何对齐和 affordance reasoning 必须被明确编码、对齐或监督，才能在复杂场景中稳定体现。

第三，后训练配方与系统工程的重要性显著上升。OpenVLA-OFT 和 LingBot-VLA 尤其说明，很多性能差异并不来自抽象意义上的“模型架构革命”，而来自更贴合目标任务的 adaptation recipe、更高效的训练栈以及更严格的评测协议。这意味着 VLA 研究正在从单论文式模型比较走向更成熟的系统优化阶段。

第四，长时任务能力开始被显式模块化。MEM 不是在原模型上简单增加更长上下文，而是把记忆拆分成短时视频记忆与长时语言记忆。这与空间显式化的过程具有同构性：一项长期被视为隐式能力的因素，一旦被证明决定任务边界，就会被架构化。

第五，humanoid whole-body 方向推动了 VLA 研究从“单一 policy 竞争”走向“完整系统竞争”。在这一分支中，谁的 backbone 更强固然重要，但 interface、controller、teleoperation recipe、data source、deployment stability 往往同样决定最终表现。LeVERB、WholeBodyVLA 与 Psi0 的共同价值，恰恰在于它们把这些系统变量一起抬到了研究正中央。

## 八、结论与对后续研究的启示

综合上述比较，可以得出一个相对清晰的结论：当前 VLA 领域的主要进步，并不是简单沿着“更大模型、更大数据、更高分数”的线性方向推进，而是在不断识别旧范式中被过度简化的问题，并将其显式重构为独立变量。动作表示、空间建模、长时记忆、whole-body 执行约束，正是这几年来被逐步显式化的四个关键变量。

若研究目标仍然是单臂桌面操作，那么当前最值得关注的不是重新发明一个完全不同的 backbone，而是如何在动作接口、空间表征与后训练配方之间做出更有针对性的组合。例如，OpenVLA-OFT 的经验说明训练配方足以改写性能边界，而 BridgeVLA 与 InternVLA-M1 则说明空间对齐与空间先验在真实操作中并非边缘问题。

若研究目标进一步指向 humanoid whole-body loco-manipulation，则本文认为一个更强的判断已经成立：单一端到端 VLA 主干并不足以自然解决问题。更合理的方向是，把 VLA 视为高层语义与决策先验的承载体，再通过 latent interface、task-aligned low-level controller、staged training、memory module 与 deployment stabilizer 等机制，将其嵌入一个完整系统之中。也正是在这一意义上，LeVERB、WholeBodyVLA 与 Psi0 不只是“新的模型”，而是在重新定义 VLA 与具身控制系统之间的分工边界。

| 工作 | 核心接口 | 相对前代的主要推进 | 当前主要局限 |
